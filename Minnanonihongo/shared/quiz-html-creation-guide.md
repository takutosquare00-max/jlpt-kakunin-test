# みんなの日本語 ユニット別 HTMLクイズ作成仕様書

## 1. 概要

各ユニットの `*-test-questions.md`（4択問題リスト）から、以下の2種類のHTMLテストファイルを生成する。

| テスト種別 | 対象問題 | 1ファイルあたり上限 | 制限時間 |
|---|---|---|---|
| 文法・文型・語彙テスト | 文法・文型問題 ＋ 意味問題 | 20問 | 15分 |
| 漢字テスト | 読み方問題 ＋ 漢字選択問題 | 上限なし（1ファイル） | 15分 |

---

## 2. 問題の振り分けルール

### 漢字テストに含める問題
- 「〜の読み方はどれですか」形式（漢字→ひらがな）
- 「〜の漢字はどれですか」形式（ひらがな→漢字）

### 文法・文型・語彙テストに含める問題
- 活用形問題（命令形・禁止形・て形・た形・ない形 など）
- 文法・文型問題（〜ほうがいい、〜と言っていました、〜ていただけませんか など）
- 意味問題（「〜の意味はどれですか」「正しい文はどれですか」など）

> **判断基準**：問題が「読み方」「書き方（漢字表記）」を問うなら漢字テスト。  
> 「文法的な使い方」「語の意味・用法」を問うなら文法・文型・語彙テスト。

---

## 3. パート分割の手順

### 3-1. 問題数を集計する

`*-test-questions.md` の解答表（`## 【解答】`）を参照し、各種別の問題数を数える。

### 3-2. パート数を決める

```
文法・文型・語彙テストのパート数 = ceil(問題数 ÷ 20)
```

均等分割（例：35問 → Part1: 18問 / Part2: 17問）を原則とする。  
ただし、セクション（問題群）の途中で分割しないほうが望ましい場合は、  
セクション境界に合わせて調整してよい（各パートが上限20問を超えなければ可）。

### 3-3. 最終パートの問題数が少なすぎる場合の調整

最終パートが極端に少ない（目安：**10問以下**）場合は、上限20問を超えてでも  
前のパートに問題を移し、各パートが均等になるよう再配分する。

**判断フロー：**

```
最終パートの問題数 ≤ 10問
  → パート数を1つ減らし、全問題を均等に再配分する
  → 各パートが20問を超えてもよい（上限は設けない）

最終パートの問題数 > 10問
  → 上限20問のまま分割する
```

**具体例：**

| 総問題数 | 機械的な分割 | 調整後 |
|---|---|---|
| 42問 | 20 / 20 / 2 | 21 / 21（パートを2つに削減） |
| 63問 | 20 / 20 / 20 / 3 | 21 / 21 / 21（パートを3つに削減） |
| 50問 | 20 / 20 / 10 | 17 / 17 / 16（パートを3つに削減） |
| 46問 | 20 / 20 / 6 | 23 / 23（パートを2つに削減） |
| 55問 | 20 / 20 / 15 | 20 / 20 / 15（最終パートが15問 > 10問なのでそのまま） |

> セクション境界を優先するため、均等に割り切れない場合は  
> 前のパートをやや多くし、最終パートをやや少なくする形でよい。

### 3-3. 漢字テストのパート数

問題数にかかわらず **1ファイル** にまとめる。

## 4. ファイル命名規則

```
unit{N}-quiz-part1.html        # 文法・文型・語彙テスト パート1
unit{N}-quiz-part2.html        # 文法・文型・語彙テスト パート2（必要な場合）
unit{N}-quiz-part{X}.html      # さらにパートが増える場合も同様
unit{N}-kanji-vocab-quiz.html  # 漢字テスト
```

`{N}` はユニット番号（例：33）。保存先は `Minnanonihongo/unit31-35/unit{N}/`。

---

## 5. HTML構造仕様

### 5-1. 全体構造

```
<header>      ← タイトル・タイマー・プログレスバー・問題数バッジ（sticky）
<div.section> ← セクションブロック（問題群ごとに1つ）
  <div.section-title>  ← セクション名（問題番号範囲を明記）
  <p.section-lead>     ← 問題形式の説明文
  <div.question> × N  ← 各問題
<button.btn-submit>   ← 採点ボタン
<div.result-panel>    ← 採点結果パネル（初期非表示）
```

### 5-2. 問題ブロックの種別

| 問題タイプ | 構造パターン |
|---|---|
| 通常の選択問題 | `.question > .q-text + .options + .explanation` |
| 会話穴埋め問題 | `.question > .dialogue + .q-text + .options + .explanation` |
| 長文選択肢（文全体が選択肢） | `.options[style="grid-template-columns:1fr"]` で1列表示 |
| 漢字の読み方問題 | `.q-text（番号のみ）+ .kanji-prompt（漢字を大きく表示）+ .options + .explanation` |
| 漢字選択問題 | `.q-text（読み方を問う文）+ .kanji-prompt（ひらがなを大きく表示）+ .options + .explanation` |

### 5-3. 選択肢のマークアップルール

- 正解の選択肢に `value="1"`、他は `value="0"`
- 解答表に複数正解（例：`b/c`、`b/d`）がある場合は、複数正解をそのまま保持し、該当するすべての選択肢に `value="1"` を付けてHTMLを作成する
- `input` の `id` と `name` の命名：
  - 文法・語彙テスト（quiz-partX）：`name="q{N}"` / `id="q{N}a"〜"q{N}d"`（アルファベットサフィックス）
  - 漢字テスト（kanji-vocab-quiz）：`name="q{N}"` / `id="q{N}1"〜"q{N}4"`（数字サフィックス）
- `.explanation` には `✅ 正解：{初期位置番号} — {解説}` の形式で記載（シャッフル後に自動更新される）

## 6. JavaScript仕様

### 6-1. 定数（ファイルごとに変更する箇所）

```javascript
const TOTAL = {問題数};   // 例：18, 17, 34
const TIME_LIMIT = 900;   // 15分固定（秒）
```

### 6-2. 共通機能（変更不要）

| 機能 | 説明 |
|---|---|
| `shuffleAllOptions()` | ページ読み込み時に全選択肢をFisher-Yatesシャッフル。正解の偏り防止ロジック付き。シャッフル後に `.explanation` の正解番号を自動更新。 |
| `startTimer()` | カウントダウンタイマー。残り5分で黄色点滅（`warning`）、残り1分で赤点滅（`danger`）。0秒になると自動採点。 |
| `submitTest()` | 全問を採点し、正解・不正解を色分け表示。解説を表示。得点率に応じてメッセージを変更。 |

## 9. よくある注意点

- **問題番号の再採番**：各HTMLファイル内では `q1` から始まる連番を使う。元のmdの問題番号とは異なる。
- **長文選択肢**：選択肢が文全体（例：間接話法・正しい文選択）の場合は `options` を `style="grid-template-columns:1fr"` で1列にする。
- **会話文**：`.dialogue` ブロックに `<span class="sp">Aさん：</span>` 形式で話者を明示する。
- **穴埋め箇所**：`<span class="blank-paren">（　　　　）</span>` を使う。
- **フリガナ**：問題文・選択肢本文（テスト対象の漢字）にはフリガナをつけない。ヘッダー・指示文には `<ruby>` タグでフリガナを付与する。
- **解説文**：`✅ 正解：X — 理由` の形式。シャッフル後に番号が自動更新されるため、初期の番号はHTMLの順番通りに記載すればよい。
