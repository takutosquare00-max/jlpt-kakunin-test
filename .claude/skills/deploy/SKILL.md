# 教材HTMLのデプロイ

引数: デプロイ対象（unit番号、ファイルパス、またはディレクトリ）。

## 手順

1. **対象特定**: 引数・会話文脈からデプロイ対象のHTMLとデプロイ先リポジトリを特定する。対応表はプロジェクトの `CLAUDE.md`（デプロイセクション）を参照
2. **コピー**: ソース → デプロイ用ディレクトリへコピー（デプロイ用ディレクトリを直接編集した場合は不要）
3. **差分確認**: `git status` / `git diff --stat` で意図した変更だけかを確認
4. **commit & push**: そのデプロイリポジトリで実行
5. **確認**: `gh run list` でPagesビルドの完了を確認（失敗時はログを見て対処）
6. **完了報告**: 返答を**「完了です」**で始め、**同時に** `open -a Safari "公開URL"` で公開ページを開く
   - part1/part2があるときは part1 を優先
   - file:// は開かない。必ず公開URL
   - HTML更新のないpush（設定・mdのみ）ではSafariを開かない

## 注意

- 複数ファイルの一括デプロイはコピー→1コミットにまとめる
- `jlpt-kakunin-test-deploy.nested-git-backup/` はバックアップなので触らない
