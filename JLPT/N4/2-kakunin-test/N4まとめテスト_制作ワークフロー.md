# JLPT N4 確認テスト「5ボリュームまとめ」制作ワークフロー

このドキュメントは、**10分×10問の確認テスト HTML を5本（ボリューム1〜5）束ねて、技能別・重複除去済みの1本のまとめテスト HTML を生成する**作業を、次回以降も同じ手順で再現するための手順書です。
現状の実装は **ボリューム1〜5 → `n4-kakunin-v1-v5-summary-50.html`（47問・30分）** です。

---

## 1. ゴールと成果物

| 項目 | 内容 |
|------|------|
| **入力** | `n4-10min-test-v1.html` … `n4-10min-test-v5.html`（各ファイル：もんだい1〜6、計10問） |
| **ビルドスクリプト** | `build_n4_v1_v5_summary.py`（スクリプトと同じフォルダを `DIR` とする） |
| **出力** | `n4-kakunin-v1-v5-summary-50.html` |
| **想定規模** | 重複除去後 **47問**、制限時間 **30分**（`TIME_LIMIT_SEC = 30 * 60`） |
| **前提** | 各 `v*` HTML は `</header>` 〜 `<button class="btn-submit"` の間に、`<div class="section">` が **6つ**（もんだい1〜6） |

---

## 2. ビルドの実行方法

```bash
cd /path/to/JLPT/N4/2-kakunin-test
python3 build_n4_v1_v5_summary.py
```

- 成功時：`Wrote .../n4-kakunin-v1-v5-summary-50.html (47 questions, removed N duplicates)` のようなログが出る。
- 依存：`beautifulsoup4`、`Python 3`（標準ライブラリに加え `bs4`）。

## 3. まとめテストの論理構造（スクリプトがやっていること）

### 3.1 セクションと技能の対応

`SECTION_SPECS` / `SEC_FIELDS` の順で固定：

1. 漢字の読み → `kanji`
2. 語彙 → `vocab`
3. 文法 → `gram`
4. 言い換え → `para`
5. 文の組み立て → `sent`
6. 読解 → `read`

### 3.2 重複除去（グローバル）

- 各 `<div class="question">` について、**読解パス・問題文（q-num除く）・各選択肢ラベル** などを正規化してフィンガープリント化し、**もんだい1→6の順、各ファイル v1→v5 の順**で初出だけ残す。
- 関数：`dedupe_globally_ordered`、`question_fingerprint`、`norm_text`。

### 3.3 通し番号と HTML 結合

- `build_merged_body` で、残った設問に **1から連番**（`renumber_question_div`）。
- v1 の各セクション冒頭にある **説明用 `<p>`**（`intro_after_title`）を、もんだい1〜5ではそのまま流用。**読解（もんだい6）だけ**は冒頭説明文に差し替え（`READING_SECTION_INTRO`）。

### 3.4 見出しの「○問」表示

- 通常セクション：`cnt = sum(count_questions_in_fragment(qh) for qh in block)`
- 読解は **2つの `.reading-bundle` フラグメント**内に計5問があるため、この集計が必要。

### 3.5 採点用 JavaScript

- `const TOTAL` が設問総数、`make_score_if_chain(counts)` で **設問番号の区間**ごとに `sec.kanji` … `sec.read` を加算。
- 選択肢は `shuffleAllChoices()` でランダム化。正解は **`value === "1"`** のラジオが正解。
- 解説文はシャッフル後、先頭の「正解：1. …」形式を正規表現で削って表示ずれを防ぐ処理あり。

### 3.6 スタイルの追加

- `</head>` の直前に `BUNDLE_CSS` を挿入し、`.reading-bundle` の境界線・余白を定義。

## 4. 読解（もんだい6）の固定仕様【重要】

通常の dedupe 結果の読解5問は **使わない**。`main()` 内で **必ず上書き**する：

```python
per_skill_html[5] = merged_reading_bundles()
counts = [
    sum(count_questions_in_fragment(qh) for qh in per_skill_html[si])
    for si in range(6)
]
```

### 4.1 記事構成（本文の出所）

| 記事 | 結合する元ファイル（各ファイルのもんだい6・先頭の1問の本文） | 設問数 |
|------|----------------------------------------------------------------|--------|
| **記事1** | `v3` + `v2` + `v4`（この順で `<br/><br/>` 区切り） | **3問**（通し 43〜45） |
| **記事2** | `v1` + `v5` | **2問**（通し 46〜47） |

- 本文抽出：`_mondai6_first_question` が各対象ファイルの読解セクション先頭の `.question` を取得し、`_reading_passage_contents` で `.reading-passage` の **内部 HTML** を取り出す。

### 4.2 HTML 構造（`.reading-bundle`）

1. `<div class="reading-bundle">`
2. 子の先頭：`<div class="reading-passage">` に **結合済み本文のみ**
3. 続けて **複数**の `<div class="question">`（本文は繰り返さない。選択肢・解説のみ）

`count_questions_in_fragment` は、ルートが `.reading-bundle` のとき **直下の** `.question` の個数を数える（`recursive=False`）。

### 4.3 設問内容（43〜47）— N4向け・独自出題

元テストの読解の「問題文・選択肢・正解」は **使わない**。`merged_reading_bundles()` 内で `_add_reading_question` により **すべて定義し直す**。

現行の正解番号（ビルド直後・シャッフル前のラベル番号）：

| 通し | 内容の要約 | 正解（表示上の番号） |
|------|------------|----------------------|
| 43 | 秋祭りの会場までの行き方 | **2** 駅から徒歩で15分です |
| 44 | 文化祭の開催日と時間 | **2** 来週の土曜日、午前10時〜午後4時まで |
| 45 | 創立記念日の月曜日 | **3** 学校は休みで、その日は創立記念日です |
| 46 | 話者が友だちと山に登った日 | **2** 先週の日曜日 |
| 47 | スーパーまでの行き方 | **1** 自転車で10分ぐらいです |

- プレースホルダ：`name="q0"`, `id="q0a"`… はビルド時に `renumber_question_div` で `q43` 等に置換される。
- 正解ラジオだけ `value="1"`、他は `"0"`。

### 4.4 読解セクションの冒頭文

- `READING_SECTION_INTRO`：記事が2本あり、1本目の後に3問・2本目の後に2問であること、同じ記事に照らして確認してよい旨を記載（ソース上は Unicode エスケープ混在可）。

### 4.5 補助関数・import

- `_add_reading_question(soup, bundle, q_text, choices, explanation)` … `bundle` は `bs4.element.Tag`。
- `from bs4.element import Tag` が必要。

## 6. 別の「5本束ね」を作るときの手順（v6〜v10 など）

1. **`build_n4_v1_v5_summary.py` をコピー**して例：`build_n4_v6_v10_summary.py`。
2. 次を一括置換・編集する：
   - `SOURCES` のレンジ：`range(1, 6)` → `range(6, 11)`、ファイル名パターン `n4-10min-test-v{i}.html` は維持。
   - `OUT`：例 `n4-kakunin-v6-v10-summary-XX.html`（実際の問数に合わせる）。
   - `page_title` / バッジ文言の「1-5」表記を新レンジに合わせる（`main` 内の `page_title` 生成）。
3. **読解を記事統合する場合**：
   - `merged_reading_bundles()` 内の **`_mondai6_first_question(DIR / "...")` のパス**を、新5本に合わせて変更する（どの v をどの段落に使うかの表をこの MD に追記すると安全）。
   - **43〜47の問題文・選択肢・正解**は、新しい本文に合わせて `_add_reading_question` の引数を**すべて**差し替える（本文と矛盾しないこと）。
4. `TIME_LIMIT_SEC` を問数・方針に合わせて調整。
5. `python3 build_n4_v6_v10_summary.py` で生成し、ブラウザで **通し番号・採点・読解2記事**を確認。

---

## 7. トラブル時のチェックリスト

- **SyntaxError / 文字化け**：長い日本語をソースに直書きすると起こることがある → **`merged_reading_bundles` 内は `\uXXXX` 形式**で書くと安全。
- **読解が5問にならない**：`count_questions_in_fragment` と `.reading-bundle` 直下の `.question` 数が一致しているか。
- **採点がずれる**：各問ちょうど1つ `value="1"` か、`counts` が `per_skill_html` と一致しているか。
- **`Tag` 型ヒント**：`_add_reading_question` の `bundle: Tag` を使う場合は `from bs4.element import Tag` が必要。
